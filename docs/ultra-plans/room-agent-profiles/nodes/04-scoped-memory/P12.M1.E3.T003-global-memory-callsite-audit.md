# P12.M1.E3.T003 Global Memory Callsite Audit

Scope: production agent/session prompt paths matching `src/agent*.ml`, `src/session*.ml`, and `src/prompt_builder.ml`.

## Result

All direct calls to `Memory.search`, `Memory.list_core`, and `Memory.recall_core` in the audited production paths are accounted for below. `src/session*.ml` and `src/prompt_builder.ml` currently contain no direct calls to these APIs.

## Legacy routing fallback

`Agent.inject_search_context` now has two explicit routes:

- Scoped route: when both `scope_kind` and `scope_key` are available, prompt context uses scoped message search plus scoped memory rows.
- Legacy fallback route: when scope metadata is absent, the agent keeps the existing global message/core-memory behavior for unprofiled or legacy sessions only.

Profiled room/thread turns must pass scope metadata and must not intentionally use the legacy fallback. The TODO markers below keep remaining global reads visible until the compaction/tool-call paths can carry scope context or be explicitly gated as admin/legacy-only behavior.

## Non-agent/session callsites intentionally outside this audit

The global memory APIs also appear in lower-level memory tests, CLI/admin command bridge code, and built-in memory tools. Those are not automatic agent/session prompt routes and are outside this task's acceptance scope; future work can separately gate them as explicit admin/legacy surfaces.
